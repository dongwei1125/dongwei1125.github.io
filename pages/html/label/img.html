            # Img

            # 概述

              ;img (opens new window) 用于将图片嵌入文档。

              ;img是行内单标签元素，同时img也是可替换元素（可修改宽高）。

            # 可替换元素

              ;HTML元素可分为可替换元素和不可替换元素，详细参考 (opens new window)。

              绝大多数的HTML元素都是不可替换元素，其内容由DOM内容决定。例如<p>hello world</p>，展示的内容就是文本节点hello world。

              而对于可替换元素，其内容则由元素的标签和属性决定。例如<img>元素的内容实际是由src属性读取的图片的原始宽高来确定的。

              典型的可替换元素有<iframe>、<video>、<embed>、<img>。其它元素仅在特定情况下为可替换元素，例如type="image"的input元素则是可替换元素，其余类型的input元素则为不可替换元素。

              可替换元素一般有内在尺寸，所以具有width和height，因此可以修改，默认显示为图片的原始宽高。

            # 属性

            # src

              指定嵌入图片的路径。

            <img src="image.png" alt="">

            # alt

              指定图像无法显示或用户禁用时的替代文本，就可访问性来说会非常有用。注意IE7及以下浏览器，鼠标悬浮至图像上时会显示alt属性值。

            <img src="" alt="橙子">

            # width

              指定图像的宽度，单位为px或者百分比。

            # height

              指定图片的高度，单位为px或者百分比，详细参考 (opens new window)。

              为图片指定width和height有很多优点，可在页面加载时为图像预留空间。

              通常不指定图片宽高，浏览器在解析时无法知晓图片的尺寸，也就无法为图像保留合适的空间，因此图像加载完成时，页面的布局就会发生变化，造成页面重排。

              而指定图片宽高，浏览器在解析时就预留了位置，图片加载完成时，浏览器则自动调整图片，使其适应此预留空间，避免了页面某些内容的移动。

            img {
  background: lightblue;
}
...

<div>
  <img src="orange.png" alt="">
  <p>hello world</p>
</div>
<div>
  <img src="orange.png" alt="" width="200px" height="200px">
  <p>hello world</p>
</div>

              如下为两种情况的对比图，视觉体验上也是第二种较好。

              注意虽然指定width和height有一定的好处，但同时指定width和height是可能会造成图片拉伸扭曲的。若要图片正常显示，可以使用标准宽高的图片或者利用CSS的 object-fit (opens new window) 来调整。

              另一个缺点是若用户禁用图片或者图片无法显示时，浏览器还是会把为图像预留的空间以指定的尺寸显示出来，造成页面空间浪费，视觉上显得大而空。

            <img src="" alt="" width="200px" height="200px">
<p>hello world</p>

              解决方式则可使用alt属性去描述图片信息，此时若图片加载失败，浏览器将只会显示图像图标和alt信息。

            <img src="" alt="橙子" width="200px" height="200px">
<p>hello world</p>

            # intrinsicsize

              提案属性已无效，用于指定图像加载的大小。

            <img intrinsicsize="250 x 200" width="" height="" src="image.png">

              此提案包含如下三种情况，详细参考 (opens new window)。

            • 未指定width和height时，图像尺寸由intrinsicsize决定
            • 指定了width时，将按照intrinsicsize的长宽比计算出高度
            • 指定了width和height时，intrinsicsize仅仅影响naturalWidth和naturalHeight的值

            # srcset

              用于提供多个图像源，以逗号分隔。

              第一种格式为图像URL和像素比描述符，若未指定默认为1x，针对不同像素比的屏幕显示不同图片。

            <img srcset="orange.png 1x, apple.png 2x, pear.png 3x">

              ;windows系统显示器默认设备像素比为1，因此将呈现orange.png。而iPhone 6像素比为2，因此将呈现apple.png。iPhone 6 Plus像素比为3，将呈现pear.png。

              另一种格式为图片URL和宽度描述符，通过计算像素比与sizes属性值的乘积呈现不同图片。

            <img srcset="orange.png 100w, apple.png 200w, beer.png 300w" sizes="100px">

              ;window系统下100*1 = 100w，将呈现orange.png。iPhone 6像素比为2，100*2 = 200w，将呈现 apple.png。而iPhone 6 Plus像素比为3，100*3 = 300w，将呈现pear.png。

            # sizes

              用于指定图像预期尺寸，以逗号分隔，格式为媒体条件和尺寸值。注意当srcset使用了宽度描述符w时，sizes才生效。

            <img srcset="orange.png 100w" sizes="(max-width: 360px) 340px, 128px">

              即当屏幕宽度小于等于360px时，sizes属性值为340px，否则为128px。

              关于sizes与srcset属性更多用法 详细参考 (opens new window)。

            # crossorigin

              指定是否为元素启用CROS模式，并定义凭据模式，启用CROS的图片可在canvas元素中重复使用，而不会被污染。

              ;crossorigin可用值如下。

            • anonymous：即匿名的意思，表示执行一个跨域请求且不会携带任何用户信息，仅根据http头校验数据是否可用
            • use-credentials：即使用凭证，表示执行一个跨域请求并携带用户信息，服务端会校验凭证信息并响应，浏览器根据响应信息判断是否采用响应数据

              注意指定crossorigin属性请求头中就会有origin字段。

              ;anonymous请求中不会携带用户信息（例如cookie），请求头中的origin需要与响应头的access-control-allow-origin字段值相匹配。

              ;use-credentials请求中会携带用户信息，而响应头中access-control-allow-origin则为具体的域名，不能为通配符*。响应头中access-control-allow-credentials为true，表示允许请求中携带凭证信息。而其它情况，不符合要求，浏览器将不会把响应内容返回。

            # ismap

              将图像定义为服务端图像映射。

              当点击服务端图像映射时，会将点击坐标以URL查询字符串的形式发送到服务器。

            <a href="http://127.0.0.1:3000">
  <img src="static/apple.png" alt="" ismap>
</a>

              点击坐标值相对于图像的左上角，若CSS指定了图片大小，则点击的坐标值也会变化。

              注意只有img元素是某个a元素的后代，ismap属性才生效。

            # loading

              指定浏览器加载图片的方式。

              属性值包括如下。

            • eager：立即加载图像
            • lazy：延迟加载图像，即懒加载，滚动距离和加载策略由浏览器定义，详细参考 (opens new window)
            <ul>
  <li>1</li>
  <li>2</li>
  <li>3</li>
  ...
</ul>
<img src="orange.png" alt="" loading='eager'>
<img src="apple.png" alt="" loading='lazy'>

            # referrerpolicy

              指定获取图片的请求的Referrer策略。

            <img src="apple.png" alt="" referrerpolicy="unsafe-url">

              关于Referer和Referter Policy，详细参考。

            # usemap

              将图像定义为客户端图像映射，与map元素的id相关联，建立img与map之间的关系。

            <img src="orange.png" usemap="#map">
<map name="map">
  <area coords="0,0,100,100" href="http://www.baidu.com">
  <area coords="100,100,200,200" href="http://www.jd.com">
</map>

              可通过Tab键高亮area区域。

            # decoding

              提示浏览器此图像的解码方式，属性值包括sync（同步解码）、async（异步解码）、auto（浏览器自主决定解码方式）。

            # importance

              指定图片资源下载的优先级，属性值包括high（高优先级）、low（低优先级）、auto（浏览器自主决定优先级）。

            # 废弃属性

              注意以下属性HTML4已经不推荐使用，HTML5中不再支持，但是目前几乎所有浏览器都支持。

            # align

              指定图像相对于周围上下文的对齐方式。

              注意不同浏览器和同一版本浏览器的不同版本对align的处理方式不一致，如下为Chrome浏览器处理方式。

            • left：左浮动，等价于float: left与vertical-align: top，注意float浮动使元素呈现类似块级元素，而vertical-align对行内元素生效，因此vertical-align: top实际不生效
            • right：右浮动，等价于float: right与vertical-align: top
            • middle：中央对齐，等价于vertical-align: -moz-middle-with-baseline
            • top：顶部对齐，等价于vertical-align: top
            • bottom：底部对齐，等价于vertical-align: baseline

            # border

              指定图像周围边框宽度。

              以下等价于border: 2px solid #000。

            <img src="orange.png" border='2'>

            # hspace

              指定图片左右外边距。

              以下等价于margin: 0 10px。

            <img src="orange.png" hspace='10'>

            # vspace

              指定图片上下外边距。

              以下等价于margin: 10 0px。

            <img src="orange.png" vspace='10'>

            # longdesc

              指定图片长描述，属性值为图片描述信息页面的URL地址。

              目前longdesc属性在HTML5有被提及到，但是仍然是移除状态。

            // index.html
<!DOCTYPE html>
<html lang="en">

<body>
  <img src="orange.png" longdesc='longdesc.html'>
</body>

</html>

// longdesc.html
<!DOCTYPE html>
<html lang="en">

<body>hello world</body>

</html>

              ;Firefox浏览器下右击图片，选择查看描述，将跳转至longdesc.html页面。

            # 占位图

              ;src源图片加载错误时，替换src为占位图。

            <img src="image.png" onerror=src="placeholder.png">

              但是上述方式会有严重缺陷，若占位图加载错误，页面将一直连续请求获取图片。

              因此需要在src被替换的同时关闭onerror事件。

            <img src="image.jpg" onerror="onerror=null; src='placeholder.png'">

              以上虽然关闭了onerror事件，但是占位图加载失败，也就没有实际意义了，最好的方式是将占位图以base64的方式引入。

            <img src="image.jpg" onerror="onerror=null; src=placeholder_svg">
